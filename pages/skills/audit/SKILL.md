# audit v1.51.0

## Purpose

Deep per-subsystem health check with structured output. Each `/audit SUBSYSTEM` call runs targeted diagnostic commands and returns a clear PASS/FAIL per check with remediation steps.

This is the first parameterized skill — it acts as a callable function: `audit(SUBSYSTEM) → report`.

## Contract

**Input:** SUBSYSTEM parameter (openclaw | gbrain | telegram | wiki-sync | litellm | /code | full)

**Output:**
```
=== AUDIT: {SUBSYSTEM} @ {timestamp} ===
✅ check-name: detail
❌ check-name: detail — REMEDIATION: ...
RESULT: N/M passed
```

**Invariant:** Each check is self-contained — it either passes or prints a specific remediation. No ambiguous "unknown" states. Audit is always read-only (no state mutation).

**CEO review reinforcement (gstack v0.18.0.0, 2026-04-17):** **REVIEW ONLY — NO CODE CHANGES.** At every STOP point in an audit (finished a subsystem, hit an ambiguous probe, ready to report) — present findings and WAIT. Do NOT jump into fix-mode on your own. Fixes require a separate decision (with AP-9 Confusion Protocol if forked). This reinforcement repeats because agents slide into implementation during review; Madi's bar is "review only when I say review."

### openclaw

```bash
echo "=== AUDIT: openclaw @ $(date '+%Y-%m-%d %H:%M') ==="

# 1. Container running + Docker healthcheck
OC_STATUS=$(docker inspect openclaw --format '{{.State.Health.Status}}' 2>/dev/null || echo missing)
[ "$OC_STATUS" = "healthy" ] \
  && echo "✅ container: healthy" \
  || echo "❌ container: $OC_STATUS — REMEDIATION: docker start openclaw; wait 60s for healthy"

# 2. Port 18789 TCP open (OpenClaw is WebSocket — never use curl http here, will get reset)
nc -z localhost 18789 2>/dev/null \
  && echo "✅ port 18789: TCP open" \
  || echo "❌ port 18789: not open — REMEDIATION: docker restart openclaw; check logs: docker logs openclaw --tail 20"

# 3. Session file size (>500KB = context overflow risk; >1MB = compaction overdue)
SESSION_DIR=$(docker exec openclaw ls /home/node/.openclaw/agents/nous/sessions/ 2>/dev/null | grep -v checkpoint | grep '.jsonl' | head -1)
if [ -n "$SESSION_DIR" ]; then
  SESSION_BYTES=$(docker exec openclaw wc -c < /home/node/.openclaw/agents/nous/sessions/$SESSION_DIR 2>/dev/null | tr -d ' ')
  [ "${SESSION_BYTES:-0}" -lt 500000 ] \
    && echo "✅ session size: ${SESSION_BYTES:-0} bytes (healthy)" \
    || echo "⚠️  session size: ${SESSION_BYTES} bytes — approaching overflow. REMEDIATION: run_task.py will auto-compact; or manually clear session file (LESSON-101)"
fi

# 4. Wiki mount readable by container
docker exec openclaw ls /opt/nous-agaas/wiki/pages/ 2>/dev/null | grep -q "." \
  && echo "✅ wiki mount: readable" \
  || echo "❌ wiki mount: not readable — REMEDIATION: chmod -R o+rX ~/nous-agaas/wiki (LESSON-094)"

# 5. Factory end-to-end (model response)
cd ~/nous-agaas
OUT=$(python3 run_task.py "Reply with exactly: AUDIT_OK" 2>&1 | tail -1)
echo "$OUT" | grep -q "AUDIT_OK" \
  && echo "✅ factory: GLM responds ($(echo "$OUT" | head -c 30))" \
  || echo "❌ factory: no response — REMEDIATION: check litellm + ZAI resource package (LESSON-099)"
```

## Anti-Patterns

- **AP-1: Don't check port 18789 with curl HTTP** — OpenClaw is a WebSocket server. Plain HTTP GET returns "connection reset by peer" even when healthy. Use `nc -z localhost 18789` (TCP check) or `docker inspect --format '{{.State.Health.Status}}'` (Docker healthcheck). (LESSON-101)
- **AP-2: Don't check LiteLLM /health without auth** — `/health` returns HTTP 401 without the master key. Always pass `-H "Authorization: Bearer $LITELLM_MASTER_KEY"` and check for HTTP 200. (discovered session 23)
- **AP-3: Don't assume ZAI error 1113 means wallet empty** — error 1113 is "Insufficient balance OR no resource package." The wallet balance may be intact; it's often the GLM-5.1 resource package (token bundle) that's exhausted. Check open.bigmodel.cn console. (LESSON-101)
- **AP-4: Don't ignore OpenClaw session file size** — the session accumulates all task history. At 127K tokens it triggers auto-compaction, but at $1.26/1M input tokens that's $0.16 per call before compaction fires. Monitor size and clear periodically. (LESSON-101)
- **AP-5: Don't run `/audit full` on every cron** — it's a slow deep check (3-10 min). Use `light-probe` for 15-min mechanical health checks.
- **AP-6: Don't trust launchctl exit=0 alone** — verify the output artifact (log file modified, state file exists). `exit=0` can mask failures if `try/except` swallows exceptions (LESSON-097 Bug C3).
- **AP-7: Don't abort on first check failure** — run all checks, report all results, let operator decide priority.
- **AP-9: Confusion Protocol — stop at ambiguous check outcomes, don't guess the grade.** (gstack v0.18.0.0, 2026-04-17) Karpathy's #1 AI coding failure mode: confidently picking the wrong interpretation at a fork. In audit scope this fires when: (a) a probe returns ambiguous status (green-ish vs degraded — is 11-min latency acceptable or a flap?); (b) partial failure should escalate to deep audit vs be noted and continued; (c) two subsystems are degraded and only one can be probed deeply in this session. Scoped rule: state both interpretations back and ASK — "Hit a fork: probe says X, could mean (A) intermittent or (B) broken. Which read should I take?" Not a blunt "confirm everything." Don't apply to routine checks where AP-1..AP-7 already cover the decision.

- **AP-11: Skill parity enumeration MUST use `-name SKILL.md` filter, not a directory glob.** (session 42, 2026-04-17) Session 42 AP-10 pt 2 run used `ls -d pages/skills/*/` → matched `pages/skills/extracted/` (VPS skill-extractor legacy dump directory, 8 raw extraction files, NO SKILL.md) — produced false-positive mismatch (`mac= vps= airW= airR=`) and triggered a spurious FAIL.
