### FP class 1 — skill-shorthand resolver mismatch (~22 of 31 broken-link entries)

### FP class 2 — bash code in code blocks parsed as wikilinks (3 entries)

`PLAN-SECRETS-MANIFEST-V1-2026-04-17.md` contains shell code:
```bash
[[ "$(uname -s)" == "Darwin" ]] || { echo "SKIP: Mac-only"; exit 0; }
```

The `[[ ... ]]` is bash conditional expression syntax inside a fenced code block. The linter parses it as 3 broken wikilinks. Linter parser doesn't skip fenced code blocks.

**Fix path (linter upgrade):** skip wikilink scan inside ` ``` ` … ` ``` ` and ` ` ` ` … ` ` ` ` regions. One regex pass. Out of scope this session.

### FP class 3 — handoff "orphans" are by-design append-only (~680 of 697)

The lint flags ~30 of the 697 orphans here, but the universe is dominated by handoff files (`pages/progress/HANDOFF-*-session-*.md`). Handoffs are timestamp-ordered append-only records — they're NOT supposed to have inbound wikilinks. The orphan check should exclude `pages/progress/HANDOFF-*` and `pages/progress/daily-factory-analysis-*` and similar dated artifacts.
